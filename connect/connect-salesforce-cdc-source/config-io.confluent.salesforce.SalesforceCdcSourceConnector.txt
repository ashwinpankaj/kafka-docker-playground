==========================
Kafka
==========================
ðŸ”˜ kafka.topic

The |ak| topic to write the Salesforce data to. This is a template driven by the data returned by Salesforce. Any field in the schema can be used but you should always pick a value that is guaranteed to exist. ``_EventType`` and ``_ObjectType`` are two metadata fields that are included on every record. For example, you could put update and deletes in a different topic by using ``salesforce.${_ObjectType}.${_EventType}``

	 - Type: STRING
	 - Default: null
	 - Importance: HIGH
	 - Required: true

ðŸ”˜ kafka.topic.lowercase

Flag to determine if the |ak| topic should be lowercase.

	 - Type: BOOLEAN
	 - Default: true
	 - Importance: HIGH
	 - Required: false

==========================
Salesforce Streaming
==========================
ðŸ”˜ salesforce.initial.start

Specify the initial starting point for the connector for replaying events. Use ``all`` to send a replayId of -2 to Salesforce that replays all events from last 24 hours, or use ``latest`` to send a replayId of -1 to Salesforce that plays only new incoming events that arrive after the connector has started. The default is ``latest`` in case there are more enqueued events than might be allowed by API limits.

	 - Type: STRING
	 - Default: latest
	 - Importance: HIGH
	 - Required: false

ðŸ”˜ salesforce.cdc.channel

The Salesforce CDC channel to subscribe to.The name should have suffix ``ChangeEvent`` for a single entity channel or ``__chn`` for a custom channel. For example ``LeadChangeEvent`` is a single entity channel and ``SalesEvents__chn`` is a custom channel. The other valid value for this config is the standard channel ``ChangeEvents``. See `Salesforce subscription channel guide <https://developer.salesforce.com/docs/atlas.en-us.change_data_capture.meta/change_data_capture/cdc_subscribe_channels.htm>`_.

	 - Type: STRING
	 - Default: null
	 - Importance: HIGH
	 - Required: true

ðŸ”˜ convert.changed.fields

Whether to convert field names within changed fields section of the ChangeEventHeader to match field names present on the Kafka record. For instance, when a nested ``Name.FirstName`` field is converted to the ``FirstName`` field on the Kafka record, setting this flag will also convert the field that is listed in ``changedFields`` within the ``ChangeEventHeader`` to be ``FirstName``, matching what is on the Kafka record.

	 - Type: BOOLEAN
	 - Default: false
	 - Importance: LOW
	 - Required: false

ðŸ”˜ salesforce.channel.entities



	 - Type: false
	 - Default: LIST
	 - Importance: List of entities in custom channel or ``ChangeEvents`` channel that the connector should process and write to Kafka.Any other entities in the channel will be ignored by the connector. This config is valid only when ``salesforce.cdc.channel`` isn't a single entity channel.
	 - Required: MEDIUM

ðŸ”˜ invalid.replay.id.behaviour

Determine if the connector should fallback to fetching all or latest events if invalid or expired replayId is provided.

	 - Type: STRING
	 - Default: all
	 - Importance: MEDIUM
	 - Required: false
