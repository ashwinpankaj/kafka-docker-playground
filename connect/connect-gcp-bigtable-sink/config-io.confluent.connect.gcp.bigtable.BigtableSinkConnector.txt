ðŸ”˜ row.key.definition

If this configuration is empty or unspecified and the Kafka Message Key is a:

	 - Type: false
	 - Default: LIST
	 - Importance: A comma separated list of Kafka Record key field names that specifies the order of kafka key fields to be concatenated to form the row key. For example the list: 'username, post_id, time_stamp' when applied to a kafka key: {'username': 'bob','post_id': '213', 'time_stamp': '123123'} and with delimiter '#' gives the row key 'bob#213#123123'. Terms nested in the key maybe also accessed using '.' as a delimiter.
	 - Required: MEDIUM

ðŸ”˜ row.key.delimiter



	 - Type: false
	 - Default: STRING
	 - Importance: The delimiter used in concatenating kafka key fields in the row key. If this configuration is empty or unspecified, the key fields will be concatenated together directly.
	 - Required: LOW
