ðŸ”˜ rejected.record.table.suffix

Suffix value for Vertica rejected records table name. With this configuration, we can provide any suffix along with a date format placeholder, for example ``_rejected_${yyyyMMdd}``. The placeholder is replaced by the current timestamp in the provided date format and is appended to the table name. By default, the value of the config is ``_rejected`` which is added as a suffix to the Vertica table name. This config is only required when rejected.record.logging.mode is set to 'TABLE'.

	 - Type: STRING
	 - Default: _rejected
	 - Importance: MEDIUM
	 - Required: false
