ðŸ”˜ gcp.firebase.snapshot

If true, the connector performs a snapshot once and listen to further changes. A complete snapshot is taken each time the connector restarts when the configuration gcp.firebase.snapshot is set to true, this may lead to connector writing duplicate records to Kafka topic. If false, the connector skips the initial database snapshot and listens to changes and writes them to kafka topic.

	 - Type: BOOLEAN
	 - Default: false
	 - Importance: HIGH
	 - Required: false
