Messsage Parsing
==========================
ðŸ”˜ cef.timestamp.mode

The strategy to use when parsing timestamps for CEF formatted messages. The default mode, 'header', will look only in the header. 'extension_field' will look in a specified extension field key, specified in a subconfig.

	 - Type: STRING
	 - Default: header
	 - Importance: LOW
	 - Required: false

ðŸ”˜ cef.timestamp.mode.extension_field.key



	 - Type: false
	 - Default: STRING
	 - Importance: The key used to extract timestamps from CEF extension fields.
	 - Required: LOW

ðŸ”˜ syslog.date.time.formatter.pattern



	 - Type: false
	 - Default: STRING
	 - Importance: The pattern to use when parsing timestamps for the Syslog messages. The default patterns are as specified in the formats supported by the Syslog connector. Alternatively, the DateTimeFormatter pattern "d-M-yy'T'HH:mm:ss" would parse the timestamp '28-2-12T09:29:10'. For more information, see the java.time.format.DateTimeFormatter documentation.
	 - Required: LOW

ðŸ”˜ syslog.time.zone

The time zone to use when parsing timestamps from syslog messages to the Kafka records. For more information, see the java.time.ZoneId documentation.

	 - Type: STRING
	 - Default: UTC
	 - Importance: LOW
	 - Required: false
